CASE STUDY 01

CollegePrep

An educational hub that eliminated academic chaos for 1700+ engineering students.

HTML CSS JavaScript SQL
The Problem
Academic information was scattered everywhere
Students juggled between Google Docs, Telegram, YouTube, and WhatsApp to find notes, track attendance, and calculate GPAs. No single source of truth. High cognitive overhead during exam season.
Target Users
Engineering students at Vellore Institute of Technology
First to final year B.Tech students — primarily mobile-first, high exam pressure, limited time. They needed a fast, organized, always-available academic resource hub.
Key Design Decisions
Reduce friction at every touchpoint
Centralized notes + video embeds eliminate tab-switching. GPA/CGPA/Attendance calculators built-in to reduce anxiety. Responsive UI ensures mobile-first access during study sessions anywhere.
UX Insight
Academic calculators as anxiety-reducers. Students obsessed over "will I pass?" — embedding GPA/Attendance calcs directly on the platform turned fear into action and drove daily active use.
Engineering Decision
WhatsApp channel as a notification layer. Instead of building a notification system, I leveraged WhatsApp's existing infrastructure — reaching 1000+ subscribers instantly, zero friction for users.
Information Architecture
Subject → Topic → Resource hierarchy. Structured navigation mirrors how students actually think about their coursework, reducing time-to-content from minutes to seconds.
Accessibility
Mobile-first, low-bandwidth aware. Platform works on 2G/3G connections common in campus areas. Lightweight embeds with fallback text ensure no student is left behind.

CASE STUDY 02

Pro Online Compiler

A VS Code-like web IDE built for students who want a real coding environment without setup.

Monaco Editor JavaScript REST API HTML/CSS
The Problem
Online compilers felt limited and unfamiliar
Most online compilers are single-file, lack themes, have ugly UIs, and bear no resemblance to real dev environments. Students using them break their flow — the tool gets in the way of thinking.
Target Users
CS students practicing coding problems
Students preparing for placements and assignments who know VS Code but can't install it in a lab or mobile context. They need familiarity, speed, and zero-setup.
Key Design Decisions
Borrow VS Code's mental model exactly
Multi-file tabs reduce context switching. Monaco Editor provides identical autocomplete and keybindings. Theme switching (dark/light) respects user preference. Download/copy features complete the "real editor" feel.
UX Research Assumption
Muscle memory matters. Students who've used VS Code for months have deep keybinding habits. Using Monaco Editor eliminated the re-learning curve and removed cognitive load instantly.
Critical Feature
Multi-tab management. Real projects aren't single files. Supporting multiple open files mirrors real engineering workflows and makes the tool usable for actual assignments — not just snippets.
Accessibility
Keyboard-first design. Every action accessible via keyboard. Theme toggle, run shortcut, and copy/download built for power users who hate reaching for the mouse.
Technical UX
Real-time execution via API. Code execution feels instant — leveraging a backend API for real language support without user-perceived latency for typical problem sizes.

CASE STUDY 03

QuickBill POS

A complete billing system that digitized small retail shops — from paper receipts to UPI QR payments.

HTML/CSS JavaScript LocalStorage QR API
The Problem
Small shops were stuck with paper bills and mental math
Small retail and service shop owners had zero digital billing infrastructure. Paper receipts were mislaid, there was no sales history, inventory was tracked in notebooks. Returns and discounts created manual errors.
Target Users
Non-tech-savvy small business owners
Shop owners aged 30–60 with minimal digital literacy. Design had to be super-intuitive — no training required. Every interaction needed to work on a basic Android phone in a busy shop environment.
Key Design Decisions
One-tap flow for every critical action
Add item → auto-calculate total → generate UPI QR → share bill via WhatsApp. The entire billing flow designed to be completed in under 30 seconds with zero technical knowledge.
Killer UX Feature
WhatsApp bill sharing = zero friction for customers. Customers don't need to install any app. A bill comes to them on WhatsApp — a platform already on every phone. This insight removed the biggest adoption barrier.
Engineering Decision
LocalStorage for offline-first resilience. Small shops have unreliable internet. Using LocalStorage ensures sales data and inventory persist even when connectivity drops — no cloud dependency needed.
UPI QR Integration
Auto-generated payment QR per transaction. QR code generated dynamically with the exact bill amount. Eliminates manual UPI amount entry errors — a major pain point in small shop transactions.
User Research Assumption
Digital literacy is low; trust in paper is high. Added printable bill functionality to bridge the trust gap — owners could still hand customers a physical receipt while operating digitally behind the scenes.
